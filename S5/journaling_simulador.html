EXT4 permite elegir entre 3 modos de journaling. Cada modo ofrece un balance diferente entre seguridad y rendimiento. Haz clic en cada tarjeta para explorar.

data=journal
Máxima seguridad
Registra metadatos Y datos completos en el journal antes de escribir. Máxima protección contra corrupción.
Seguridad
Rendimiento
Caso de uso: Bases de datos financieras, registros médicos, sistemas donde la integridad es crítica.
data=ordered
Equilibrado (default)
Registra solo metadatos en el journal. Los datos se escriben ANTES que sus metadatos, garantizando consistencia.
Seguridad
Rendimiento
Caso de uso: Servidores web, file servers, aplicaciones de uso general. Es el modo por defecto de la mayoría de distribuciones Linux.
data=writeback
Máximo rendimiento
Solo registra metadatos. Los datos pueden escribirse en cualquier orden respecto a los metadatos. Máxima velocidad.
Seguridad
Rendimiento
Caso de uso: Logs temporales, cachés, datos regenerables. Máximo throughput de escritura donde la pérdida parcial de datos es aceptable.
Comandos para configurar el modo
$ sudo mount -o data=journal /dev/sda1 /mnt
$ cat /proc/mounts | grep sda1 # Ver modo actual
$ sudo tune2fs -o journal_data /dev/sda1 # Cambiar default
