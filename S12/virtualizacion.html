01 ──────
Full Virtualization
Emulación de Hardware

El hipervisor emula hardware completo. El sistema invitado no sabe que está virtualizado.

VENTAJA ↑Alta compatibilidad
DESVENTAJA ↓Menor rendimiento
// EJEMPLOS
VMware ESXi antiguo KVM QEMU AWS Nitro
RENDIMIENTO RELATIVO
COMPATIBILIDAD

El hipervisor intercepta cada instrucción privilegiada del OS invitado y la emula en software. Esto introduce latencia adicional pero permite ejecutar cualquier OS sin modificaciones.

AWS Nitro es el caso más moderno: combina hardware personalizado (tarjetas NitroCards) con un hipervisor minimalista para recuperar casi todo el rendimiento perdido.

02 ──────
Paravirtualización
Paravirtualization

El OS invitado está modificado para comunicarse directamente con el hipervisor mediante hypercalls, reduciendo overhead.

VENTAJA ↑Muy eficiente
DESVENTAJA ↓Kernel especializado
// EJEMPLOS
Xen UML
RENDIMIENTO RELATIVO
COMPATIBILIDAD

En lugar de emular hardware, el OS invitado llama directamente al hipervisor (hypercalls). Esto elimina la sobrecarga de intercepción pero exige recompilar el kernel del OS invitado.

Xen es el referente histórico: Amazon EC2 usó Xen hasta migrar progresivamente a Nitro. UML (User-Mode Linux) es más académico, usado para sandboxing y enseñanza.

VER MÁS ▼
03 ──────
Contenedores
OS-level Virtualization

No virtualiza hardware. Aísla el espacio de usuario del OS mediante namespaces y cgroups. Comparten el mismo kernel.

VENTAJA ↑Ligero · Rápido
DESVENTAJA ↓Mismo kernel
// EJEMPLOS
LXC Docker Kubernetes
RENDIMIENTO RELATIVO
COMPATIBILIDAD

Los contenedores usan namespaces (PID, red, mount, UTS, IPC) para aislar procesos y cgroups para limitar recursos (CPU, RAM, I/O).

No pueden correr un kernel distinto al del host. Esto los hace inadecuados para aislar cargas con kernels heterogéneos, pero ideales para microservicios y CI/CD.

04 ──────
Asistida por Hardware
Hardware-Assisted Virtualization

Usa extensiones del CPU (Intel VT-x / AMD-V) para crear un "Ring -1" donde el hipervisor opera con privilegios máximos acelerando el acceso al hardware.

VENTAJA ↑Rendimiento near-native
DESVENTAJA ↓Depende del HW
// EJEMPLOS
VMware ESXi moderno KVM Proxmox VE
RENDIMIENTO RELATIVO
COMPATIBILIDAD

Intel VT-x (Virtualization Technology) y AMD-V (AMD Virtualization) añaden un nuevo nivel de privilegio (VMX root / Ring -1) que permite al hipervisor controlar directamente las instrucciones privilegiadas sin emulación.

Proxmox VE combina KVM (hardware-assisted) con LXC (contenedores), dando lo mejor de ambos mundos en el mismo clúster — exactamente la arquitectura de tu homelab.
