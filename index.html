BIOS
BASIC INPUT/OUTPUT SYSTEM • 8-BIT ERA
▸ OPERACIÓN POST (POWER-ON SELF-TEST)
01
RESET VECTOR → CPUProcesador salta a dirección física 0xFFFF0 en memoria ROM. Primer instrucción ejecutada tras encendido.
02
VERIFICACIÓN DE RAMEscritura y lectura en cada celda de memoria para detectar fallas (Interrupt 17h → controlador de memoria).
03
INICIALIZACIÓN VGAINT 10h activa controlador de video. Muestra POST code en pantalla. Modo 16-bit Real Mode exclusivo.
04
DETECCIÓN DE DISCOBus IDE/ATA interrogado vía I/O port 0x1F0. Lee sector MBR (512 bytes, firma 0x55AA).
05
CARGA BOOTLOADERMBR copiado a 0x7C00 en RAM. CPU transfiere control: jmp 0000:7C00 → OS toma el mando.
▸ CARACTERÍSTICAS TÉCNICAS
BIOS
Modo: 16-bit Real
ROM: 256KB–1MB
Interface: Texto
Boot: MBR (2TB max)
Red: No nativa
Seguridad: Nula
Init time: ~30s
UEFI
Modo: 32/64-bit
Flash: 4MB–32MB
Interface: GUI
Boot: GPT (8ZB max)
Red: PXE nativo
Seguridad: Secure Boot
Init time: ~5s
UEFI
UNIFIED EXTENSIBLE FIRMWARE INTERFACE • INTEL 2005
▸ FASES DE INICIALIZACIÓN UEFI
SEC
SECURITY PHASECPU resetea en modo real. UEFI verifica integridad de firmware con hash SHA-256 antes de ejecutar cualquier código.
PEI
PRE-EFI INITInicializa controlador de memoria (XMP/EXPO), configura PCIe lanes. Pasa HOBs (Hand-Off Blocks) a siguiente fase.
DXE
DRIVER EXECUTION ENVCarga drivers EFI de GPU (GOP), NVMe, USB 3.x, Red (PXE). Protocolo EFI_BLOCK_IO para cada dispositivo.
BDS
BOOT DEVICE SELECTLee variables NVRAM (boot order). Detecta partición ESP (FAT32) con bootloader EFI (BOOTX64.EFI). Secure Boot valida firma.
RT
RUNTIME SERVICESGetTime, SetVariable, ResetSystem siguen disponibles para el OS. Windows/Linux acceden vía syscall directo.
AL
AFTER LIFEExitBootServices() llamado por OS kernel. UEFI libera memoria. Solo Runtime Services permanecen activos.
